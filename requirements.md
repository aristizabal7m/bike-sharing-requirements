# SISTEMA DE PRESTAMO DE BICICLETAS


## Stakeholders

- Estudiantes
- Administración de la universidad

## Requisitos Funcionales

- 1RF: El sistema debe recibir datos de prestamista.
- 2RF: El sistema debe permitir reservar una bicicleta.
- RF3: El sistema debe permitir registrar la devolución de una bicicleta.
- RF4: El sistema debe permitir a los estudiantes registrar la devolución de una bicicleta.
- RF5: El sistema debe permitir a los usuarios reportar problemas o daños en las bicicletas.
- RF6: El sistema debe mostrar la ubicación de las estaciones de bicicletas dentro del campus.
- RF7: El sistema debe permitir al equipo de mantenimiento consultar las bicicletas reportadas confallas.
- RF8: El sistema debe registrar el historial de préstamos realizados por cada estudiante.

## Requisitos No Funcionales
